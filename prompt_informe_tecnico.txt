Contexto Detallado del Sistema:
1. Arquitectura: 
   - Backend Lógico: C# (Clases Cubo, Cara, Nodo, Grafo, Movimiento).
   - Frontend Interactivo: JavaScript (Three.js para el motor 3D y Vis-network para el grafo visual).
2. Conceptos de Programación Orientada a Objetos (POO):
   - Herencia y Clases Abstractas: La clase 'Solver' define la firma del método 'Resolver()'. Las clases SolverBFS, SolverDFS, SolverBacktracking y SolverHeuristico heredan de esta base.
   - Polimorfismo: El sistema trata a todos los algoritmos como un tipo 'Solver', permitiendo cambiar de estrategia en tiempo de ejecución.
   - Encapsulamiento: El estado interno del cubo (matrices de colores) es privado, accesible solo mediante métodos controlados.
   - Composición: La clase Cubo se compone de 6 objetos Cara, y cada Cara contiene una matriz de enumeraciones ColorCubo.
3. Estructuras de Datos y Algoritmos:
   - Grafos de Estados: Cada combinación de colores es un vértice. Cada giro es una arista con un peso uniforme de 1.
   - BFS (Breadth-First Search): Usa una Cola (Queue) para encontrar el camino óptimo (mínimos movimientos).
   - DFS (Depth-First Search): Usa una Pila (Stack) con profundidad limitada a 6 para evitar explosión combinatoria.
   - Backtracking: Implementación recursiva que deshace movimientos para explorar ramas alternativas.
   - A* (Heurística): Usa una Cola de Prioridad (PriorityQueue) y una función F(n) = G(n) + H(n). La heurística H(n) calcula la distancia de cada pieza a su posición correcta (Sticker mismatch).
   - Eficiencia: Uso de Tablas Hash (HashSet) para almacenar estados visitados (claves tipo string) y evitar ciclos infinitos.
